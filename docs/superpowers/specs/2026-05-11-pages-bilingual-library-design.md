# GitHub Pages 双语资料库设计

日期：2026-05-11
状态：已确认设计，等待用户审阅书面 spec

## 背景

当前仓库是一个网球相关 PDF 收藏，`docs/index.html` 已作为 GitHub Pages 入口，但页面仍是深色主题的基础书目列表。新的页面需要同时服务三类用途：给自己和朋友快速下载资料，给公开访客了解收藏内容，并给网球学习者按主题找到合适资料。

用户已确认的方向是“现代资料库”：浅色现代产品页、中英语言切换、资料卡片、小型资料库式搜索筛选，以及 Raw PDF 直链下载。

## 目标

- 将 `docs/index.html` 优化为单页静态资料库，不引入构建步骤。
- 提供中文和英文两套界面文案，通过顶部语言切换控制显示。
- 添加清晰简介，说明资料库内容覆盖网球技术、专项训练、比赛心理、名将传记和运动科学。
- 将书目从纯列表升级为资料卡片，包含标题、分类、标签、简介和下载按钮。
- 提供关键词搜索和分类筛选，帮助访客快速定位资料。
- 保持 Raw PDF 直链作为主要下载方式。
- 在桌面和移动端都保持清晰、轻量、可读。

## 非目标

- 不新增后端、数据库、构建系统或框架。
- 不做用户登录、收藏、在线阅读进度或下载统计。
- 不改变 PDF 文件位置、文件名或仓库结构。
- 不对 PDF 链接做运行时可用性检测。
- 不在本次设计中制作完整封面图、OCR 全文搜索或逐页内容索引。

## 页面架构

页面继续由 `docs/index.html` 独立承载，使用内联 CSS 和少量原生 JavaScript。整体结构如下：

1. 顶部导航：品牌名称、简短英文副标题、资料数量、语言切换。
2. 首屏简介：中英可切换的资料库介绍、主要行动入口、资料统计和学习入口标签。
3. 工具栏：关键词搜索框、分类筛选按钮、当前结果数量。
4. 书目网格：响应式卡片列表，桌面端多列，移动端单列。
5. 空状态：搜索或筛选无结果时提示用户调整关键词或分类。
6. 页脚：GitHub Pages 托管说明、版权和个人学习用途提醒。

## 数据模型

书目数据保存在页面脚本中的数组里。每条记录包含：

- `file`：原始 PDF 文件名，用于生成 Raw 下载链接。
- `title.zh` / `title.en`：中英文展示标题。
- `category`：标准分类键，例如 `technique`、`training`、`biography`、`body`、`rules`。
- `tags.zh` / `tags.en`：中英文标签数组。
- `summary.zh` / `summary.en`：中英文一句简介。
记录可额外包含 `audience.zh` / `audience.en`，用于说明适合的读者或学习阶段；若没有可靠依据，可以省略。

简介需要通过抽样查看 PDF 内容来写，优先使用目录、封面、前言或章节标题确认主题。无法可靠确认细节时，简介应保守描述资料主题，避免写成明确承诺。

## 交互设计

语言切换：

- 顶部提供中文和 English 两个选项。
- 切换后更新页面简介、按钮、分类名、标签、结果计数、空状态和书目简介。
- 使用 `localStorage` 记住上次语言选择；默认语言为中文。

搜索：

- 搜索范围包括中英文标题、原始文件名、分类名、标签和简介。
- 输入时即时更新结果，不需要提交按钮。
- 搜索逻辑大小写不敏感，并兼容中文关键词。

分类筛选：

- 提供“全部”、技术、训练、传记、身体与运动科学、规则/测评等分类。
- 分类按钮展示当前选中状态。
- 搜索和分类筛选可以叠加。

下载：

- 每张卡片提供主要按钮“下载 PDF”或 “Download PDF”。
- 链接继续使用 `https://raw.githubusercontent.com/<owner>/<repo>/<branch>/<encoded file>`。
- 下载链接保留 `rel="noopener noreferrer"`，并使用正确编码处理中文文件名。

## 视觉设计

整体采用浅色现代资料库风格：淡灰绿色页面背景、白色内容卡片、细边框和网球绿主色。少量蓝色、琥珀色等辅助色用于区分主题标签，但页面不能变成单一绿色调。

首屏不做大幅英雄图片或营销式装饰，核心视觉信号来自清晰标题、简介、统计和资料卡片。按钮和卡片保持小圆角、紧凑间距和可扫描的层级。移动端重点避免文字挤压，工具栏允许换行，卡片单列显示。

## 错误和边界处理

- JavaScript 正常运行时渲染完整资料库。
- JavaScript 不可用时，页面保留基础标题、简介和 `noscript` 提示。
- 搜索或筛选无结果时展示空状态，不留下空白区域。
- 下载链接失败时不做额外网络重试或检测，保持静态站简单性。
- 如果某本书缺少英文简介，不能展示临时文案；实现前必须补齐或改为保守英文描述。

## 验证计划

- 本地打开 `docs/index.html`，确认首屏、搜索栏、分类按钮和书目卡片正常显示。
- 检查中文和英文切换，确认页面级文案、结果数量、分类名、标签和卡片简介都跟随切换。
- 搜索中文词、英文词和文件名片段，确认匹配结果正确。
- 切换多个分类，确认筛选和搜索叠加逻辑正确。
- 在窄屏宽度检查导航、首屏、工具栏和卡片没有文字重叠。
- 抽查至少 3 个 Raw PDF 链接，确认 URL 编码格式正确。

## 实施边界

本设计只要求修改 GitHub Pages 页面及其相关静态文案。PDF 文件本身、仓库根目录书目文件和下载路径保持不变。实现计划应先补齐书目元数据，再重构页面渲染和交互，最后做本地视觉与行为验证。
